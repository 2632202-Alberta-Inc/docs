title: "Ticket Status"
---

## Overview

Ticket status tracks the state of a ticket through its lifecycle, from creation to final verification, ensuring proper data quality and workflow management. The status transitions follow a defined workflow with both automated and human-triggered state changes.

## Attributes

<ResponseField name="INCOMPLETE" type="System Triggered">
  Default initial status when ticket is created. Indicates required data is
  missing.
</ResponseField>

<ResponseField name="PENDING_REVIEW" type="System Triggered">
  All required fields are present and validated; system automatically
  transitions ticket to this status.
</ResponseField>

<ResponseField name="DISPUTED" type="Human Triggered">
  Can only be set after `PENDING_REVIEW`. Indicates issues were found during
  review that require investigation.
</ResponseField>

<ResponseField name="APPROVED" type="Human Triggered">
  Can only be set after `PENDING_REVIEW`. Indicates data has been verified by
  authorized personnel for completeness and is ready for production accounting
  verification.
</ResponseField>

<ResponseField name="CLOSED" type="Human Triggered">
  Can only be set after `PENDING_REVIEW`. Indicates final verification by
  production accounting is complete. Ticket becomes immutable.
</ResponseField>

<ResponseField name="CANCELED" type="Human Triggered">
  Ticket has been voided but retained for audit purposes. Can be triggered at
  any point in the workflow.
</ResponseField>
