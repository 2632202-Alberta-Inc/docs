## Overview

A facility is a representation of a physical location that is unique by their `regulator_id` (e.g. ABBT0030001), `jurisdiction` (e.g. Alberta), and `regulator_system` (e.g. Petrinex).

## Types

The main facility types include:

- **Terminal**: A facility for the transfer of products.
- **Station**: A location for processing or storing products.
- **Plant**: A facility for processing raw materials into finished products.
- **Battery**: A facility for the collection and storage of oil and gas.
- **System**: A network of interconnected facilities.

While Flux classes the following as lines, they may be used interchangeably with facilities:

- **Rail Line**: A conduit for transporting products by rail.
- **Pipeline**: A conduit for transporting products by pipeline.

### Identification

<ResponseField name="ID" required>
  The primary key identifier used inside the Flux system (references `id` in
  facility models)
</ResponseField>

<ResponseField name="Name" required>
  Name of the facility
</ResponseField>

### Location

<ResponseField name="Latitude">
  Geographic latitude coordinate of the facility
</ResponseField>

<ResponseField name="Longitude">
  Geographic longitude coordinate of the facility
</ResponseField>

### Legal Land Description

<ResponseField name="Legal Subdivision (LSD)" required>
  First component of legal land description - divides a section into 16 equal
  parts
</ResponseField>

<ResponseField name="Section" required>
  Second component of legal land description - one square mile (640 acres)
</ResponseField>

<ResponseField name="Township" required>
  Third component of legal land description - a row of 36 sections
</ResponseField>

<ResponseField name="Range" required>
  Fourth component of legal land description - a column of townships
</ResponseField>

<ResponseField name="Meridian" required>
  Fifth component of legal land description - reference line for measuring
  ranges
</ResponseField>

### Classification

<ResponseField name="Type" required>
  Type of facility (references `id` in FacilityType model)
</ResponseField>

<ResponseField name="Sub Type" required>
  Sub-classification of the facility type (references `id` in FacilitySubType
  model)
</ResponseField>

<ResponseField name="Start Date" required>
  Date when the facility began operations
</ResponseField>

### Operations

<ResponseField name="Operator" required>
  Entity that operates the facility (references `id` in Entity model)
</ResponseField>

<ResponseField name="Operational Status" required>
  Current operational status of the facility (e.g. ACTIVE, SUSPENDED, ABANDONED)
</ResponseField>

<ResponseField name="Operator Start Date" required>
  Date when the current operator began operating the facility
</ResponseField>

### Licensing

<ResponseField name="Licensee">
  Entity holding the facility license (references `id` in Entity model)
</ResponseField>

<ResponseField name="License Status">
  Current status of the facility license (e.g. ISSUED, SUSPENDED, CANCELLED)
</ResponseField>

<ResponseField name="License Type">
  Type of license issued for the facility
</ResponseField>

<ResponseField name="License Number">
  Unique identifier assigned to the facility license
</ResponseField>

<ResponseField name="License Issued Date">
  Date when the current license was issued
</ResponseField>

### Regulatory

<ResponseField name="Regulator ID" required>
  Unique identifier assigned by the regulatory system (e.g. Petrinex ID)
</ResponseField>

<ResponseField name="Regulator System" required>
  Regulatory system that oversees the facility
</ResponseField>

<ResponseField name="Jurisdiction" required>
  Regulatory jurisdiction where the facility is located
</ResponseField>
